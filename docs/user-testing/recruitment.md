# Recruitment Guide

## Target participants

| Criterion | Preferred |
|-----------|-----------|
| Count | 3–5 per round |
| Background | GIS student, analyst, or practitioner |
| Experience | At least one semester GIS coursework or 6+ months using GIS software |
| Tools | Familiar with vector data (shapefile/GeoJSON) conceptually |
| Exclude | Authors of this thesis project; team members who built the UI |

## Screening questions

1. Have you used desktop or web GIS (e.g., ArcGIS, QGIS) in the last year?
2. Have you uploaded or edited vector geospatial data (points/lines/polygons)?
3. Are you comfortable speaking in English during a 60-minute session?

**Include** if Q1 or Q2 is yes and Q3 is yes.

## Recruitment channels

- University GIS / geography labs
- GIS LinkedIn or Discord communities (with moderator approval)
- Colleagues (non-builders)

## Email template

**Subject:** Usability study — geospatial data quality tool (45–60 min)

Hello,

I am conducting a short usability study for a master's thesis project: a web tool that validates GeoJSON/shapefile data and suggests corrections. Sessions take **45–60 minutes** (remote or in-person) and involve completing realistic tasks while thinking aloud.

**You would:**

- Upload a sample dataset and use a dashboard to review issues
- Approve/reject suggested fixes and export a report

**Requirements:** Basic familiarity with GIS data; laptop with Chrome/Edge/Firefox.

**No compensation** / **[€X gift card]** — *[edit as applicable]*

If interested, reply with your availability and whether you have used GIS software in the past year.

Thank you,  
[Your name]  
[Institution]

## Scheduling

- Allow 15 minutes between sessions for notes.
- Send participant brief + file path 24 hours ahead.
- Use consistent browser (recommend Chrome) across participants.
